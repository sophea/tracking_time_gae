Date:2017-01-13 14:56:31 

This component service is about to manage security and user management flow and it can apply with GAE and none-GAE Platform. 

There are 3 types of Authentication we currently support :

1- oAuth2 Client Credentials flow (Minimum that every App must use & Third Party Systems that integrate into backend)
2- oAuth2 Resource Owner Password flow (Username, password) - To use this type, it must pass access_token from 1- oAuth2 Client Credentials  
3- oAuth2 Authorization Code (Social Integration, e.g., Facebook, Twitter, etc.)

    • POST /admin/oauth2/v1/user
      ROLE_ADMIN ROLE_BACKOFFICE_ADMIN create user type client credentials

      Description :

      Designed for admin user who has ROLEs ROLE_ADMIN and ROLE_BACKOFFICE to create new user with type client_credentials. This new user is used by 3rd-party app. Successful response will be httpStatus OK. The created user will have role ROLE_USER, ROLE_USER_CREDENTIALS.

  • /oauth2
    • POST /oauth2/v1/user
      ROLE_USER_CREDENTIALS create a new user

      Description :

      Registers new user as user type resources owner password . when user calls this API, the user will have two ROLEs: ROLE_USER and, ROLE_USER_PASSWORD.

      If there are some extra fields to store as well such as first name, last name, date of birth and others. Your backend needs to have one listener implementation. Extra fields will manage in backend project
      The user-security-mgmt will store only username and password.

      There are 2 methods in this listener ( pre & after Process).

      1 -preProcess() : may be used for validation request params or other purposes
      2 -afterPorcess() : save your extra fields into other entity (Profile entity) of backend project.

      How to integration from backend project :

      • create a new Listener implemented (UserAccountListener) from UserListener
      • implementation preProcess & afterProcess methods
      • declare UserAccountController as autoWired tag in MvcConfig or you can do in xml configuration
      • set listener in securityModule() method: userAccountController.setListener(new UserAccountListener());
      • see more details in template-gae-backend or template-backend

      When activation account feature is used, the response in json format will not expose accessToken value at all.

      Content-Type: application/x-www-form-urlencoded

      Content-Type: application/x-www-form-urlencoded charset=utf-8

      Param Type Name Type Required Default Value Description
      query param username String X Username for login value is email address
      query param password String X Password for login
      query param appType Long Application type which is used to generate deep link send in email content to activate account.
      This param depends on state param value. If state is pending then appType has to present in the request.
      If your project does not need feature activate account please ignore this param and state param. Here are possible Values:
      • 1 - for iOS
      • 2 - for Android
      • 3 - for web

    • POST /oauth2/token
      ROLE_USER authenticate with oAuth2 Resource Owner Password flow

      Description :

      Verifies user account login with type resources owner password.
      To pass this request you need to pass authentication by user type credentials first otherwise response back 401.
      There is one listener added in this API, there are 2 two methods (pre & after Process) to be invoked.

      1 -preProcess() : may be used for validation request params or other purposes
      2 -afterPorcess() : after login successfully, this method will be called and backend project

      If you need to enable this listener, you may implement this listener in your backend integration.
      How to integration from backend project :

      • create a new Listener ex: (Oauth2LoginListener) implemented from UserListener
      • implement on preProcess & afterProcess methods
      • declare OAuth2Controller as autoWired tag in MvcConfig or you can do in xml configuration
      • set listener in securityModule() method: OAuth2Controller.setListener(new Oauth2LoginListener());
      • see more details in template-gae-backend or template-backend
      This feature is available from version 2.0.44 to up In case your application uses non standard authentication identity (phone, phoneId, ..) listener could "resolve" it by putting "realIdentity" parameter into request. This is how your listener implementation could look like 
              public void preProcess(HttpServletRequest request, String actionName) {
            if ("loginWithUserResourcesOwnerType".equals(actionName)) {
                String customIdentity = request.getParameter("username");
                if (isPhone(username)) {
                    String username = profileService.getUsernameByPhone(custmoIdentity);
                    request.setAttribute("realIdentity",username);
                } else if (isTwitterId(username)) {
                    String username = profileService.getUsernameByTwitter(custmoIdentity);
                    request.setAttribute("realIdentity",username);
                }
            }
        }
      After that realIdentity would be used as a username

null - if number of values is impossible to determine, or it's not necessary according to logic. But client can analyze hasNext parameter.
In case if it's necessary to do strict pagination with number of pages, with switching between pages, etc. then that function can implement that functionality.

reverseCursor java.lang.String This value is for providing reverse cursor required for backward pagination. In most cases this value makes sense only if it's returned in initial request, for example to provide an ability for retrieving new items created after initial (first) request, i.e. backward from 0 page
In some cases this value can be returned in all responses, not only in initial one, if business logic suppose that, in order to provide reverse pagination from every position, let's say from 4th or 5th pages etc.
